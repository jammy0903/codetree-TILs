## 1. CPU Scheduling

### 공통 공식 (모든 스케줄링 알고리즘 동일)

```
Waiting Time (WT)         = Finish Time - Arrival Time - Burst Time
Turnaround Time (TAT)     = Finish Time - Arrival Time
Response Time (RT)        = First CPU Assigned Time - Arrival Time

Average Waiting Time      = Sum of WT / Number of Processes
Average Turnaround Time   = Sum of TAT / Number of Processes
```

### HRN 전용 우선순위 공식

```
HRN Priority = (Waiting Time + Burst Time) / Burst Time
→ 값이 클수록 먼저 실행
```

### 스케줄링 알고리즘 정리

| 알고리즘 | 선점 여부 | 선택 기준 |
|---------|---------|---------|
| FCFS | 비선점 | 도착 순서 |
| SJF | 비선점 | Burst Time 짧은 것 |
| SRT | 선점 | 남은 Burst Time 짧은 것 |
| RR | 선점 | Time Quantum 단위로 순환 |
| Priority | 선점/비선점 | 우선순위 높은 것 |
| HRN | 비선점 | (WT + BT) / BT 큰 것 |

### 풀이 순서 (간트 차트)

```
Step 1. 도착시간 기준 정렬
Step 2. 알고리즘에 따라 간트 차트 작성
Step 3. 각 프로세스별 Finish Time 확인
Step 4. WT = FT - AT - BT
Step 5. 평균 WT = Sum(WT) / n
```

## 2. Cyclomatic Complexity (순환 복잡도)

```
V(G) = E - N + 2P

E: 간선(Edge) 수
N: 노드(Node) 수
P: 연결된 컴포넌트 수 (보통 1)

→ 단순화: V(G) = 결정 포인트(분기) 수 + 1
```

**결정 포인트 = if, while, for, case 등 분기문**

## 3. LOC / Man Month

```
LOC 예측값 = (낙관치 + 4×기대치 + 비관치) / 6

Man Month (MM) = LOC / 1인당 월 생산 LOC
개발 기간     = MM / 투입 인원
투입 인원     = MM / 개발 기간
생산성        = LOC / MM
```

## 4. Page Replacement (페이지 교체)

**목표: Page Fault(페이지 부재) 횟수 계산**

| 알고리즘 | 교체 기준 |
|---------|---------|
| FIFO | 가장 먼저 들어온 페이지 교체 |
| LRU | 가장 오래전에 사용된 페이지 교체 |
| LFU | 사용 빈도가 가장 낮은 페이지 교체 |
| OPT | 앞으로 가장 오래 사용 안 될 페이지 교체 (이론값) |

```
Page Fault = 프레임에 없는 페이지 요청 시 발생
Page Hit   = 프레임에 이미 있는 경우 (교체 없음)
```
