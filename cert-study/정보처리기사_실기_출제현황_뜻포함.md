## 9. 2020~2025 출제 패턴 분석 및 2026 쪽집게 전략

### 9-1. 출제 패턴 핵심 결론

2020~2025년 기출을 분석한 결과, 출제자는 **두 가지 전략을 병행**한다.

1. **반복 출제 전략** — 핵심 빈출 개념은 매년 반복해서 답으로 등장 (디자인패턴 GoF, 결합도/응집도, 커버리지, OSI 7계층, SQL 기본 문법, 관계대수 기호 등)
2. **빈칸 메우기 전략** — 같은 카테고리 내에서 이미 출제된 항목은 피하고, 아직 답으로 나오지 않은 항목을 골라 출제하는 경향 → **이것이 쪽집게 포인트**

즉 "매년 나온 것만 또 나오는" 것도 아니고, "완전히 랜덤"도 아니다. **카테고리별 구멍을 메꾸려는 움직임**이 뚜렷하므로, 미출제 항목 중 같은 군에서 다수가 이미 출제된 카테고리를 우선 학습해야 한다.

### 9-2. 최우선 쪽집게 (⭐⭐⭐ 강력 출제 예상)

카테고리 내 대부분이 출제되어 **남은 항목이 다음 타자일 가능성이 매우 높은** 핵심 구멍들이다.

| 영역 | 찍어야 할 항목 | 뜻 요약 | 출제 예상 근거 |
|---|---|---|---|
| 디자인 패턴 | Strategy | 알고리즘군을 캡슐화해 교체 가능하게 만드는 행위 패턴 | GoF 23개 중 대부분 나왔고 남은 빈칸 |
| 디자인 패턴 | Decorator | 객체에 기능을 동적으로 덧붙이는 구조 패턴 | 구조 패턴 중 유일하게 자주 묻는 건데 미출제 |
| 디자인 패턴 | Facade | 복잡한 하위 시스템을 단순 인터페이스로 감싸는 구조 패턴 | 출제 빈도 높은 개념인데 미출제 |
| 디자인 패턴 | Builder | 복잡한 객체를 단계적으로 생성하는 생성 패턴 | 생성 패턴 빈칸 메우기 |
| 디자인 패턴 | Prototype | 기존 객체를 복제해 새 객체를 만드는 생성 패턴 | 생성 패턴 빈칸 메우기 |
| SQL 조인 | INNER JOIN | 양쪽에 값이 일치하는 행만 결합하는 조인 | CROSS/UNION만 나왔는데 기본형이 안 나온 것이 이상 |
| SQL 조인 | NATURAL JOIN | 동일 컬럼을 자동 기준으로 결합하는 조인 | 조인 기본형 빈칸 |
| SQL 조인 | LEFT/RIGHT OUTER JOIN | 한쪽 테이블을 전부 포함하는 외부 조인 | 조인 기본형 빈칸 |
| 집합 연산 | INTERSECT | 두 결과 집합의 공통 행을 반환 | UNION만 나옴 |
| 집합 연산 | MINUS | 한 집합에서 다른 집합을 뺀 결과 | UNION만 나옴 |
| 정규화 | 1NF | 모든 속성이 원자값만 갖도록 분해한 정규형 | 2NF/3NF는 나왔는데 1NF가 미출제인 건 매우 이례적 |
| 정규화 | BCNF | 모든 결정자가 후보키가 되도록 하는 강한 3NF | 정규형 빈칸 메우기 |
| 트랜잭션 ACID | Consistency | 트랜잭션 전후에도 DB 일관성이 유지되는 성질 | A, I는 답으로 나왔고 C, D는 지문만 — 이제 답 차례 |
| 트랜잭션 ACID | Durability | 트랜잭션 완료 결과가 영구 저장되는 성질 | A, I는 답으로 나왔고 C, D는 지문만 — 이제 답 차례 |
| OSI 계층 | 전송 계층 | 종단 간 신뢰성 있는 데이터 전송을 담당하는 4계층 | 물리~표현까지만 나옴 |
| OSI 계층 | 세션 계층 | 통신 세션을 설정·관리·종료하는 5계층 | 물리~표현까지만 나옴 |
| OSI 계층 | 응용 계층 | 사용자 응용 서비스에 가까운 최상위 7계층 | 물리~표현까지만 나옴 |
| 보안 공격 | XSS | 웹 페이지에 악성 스크립트를 주입해 실행시키는 공격 | 대표 웹 공격인데 미출제 |
| 보안 공격 | CSRF | 사용자의 의도와 무관한 요청을 위조해 실행시키는 공격 | 대표 웹 공격인데 미출제 |
| 보안 공격 | Ransomware | 파일을 암호화하고 금전을 요구하는 악성코드 | 사회적 이슈 대비 미출제 |

| 해시 | SHA-256 | 256비트 해시값을 생성하는 SHA-2 계열 함수 | MD5만 나옴 |

| 보안 3요소 | 무결성 | 정보가 위변조되지 않고 정확성을 유지하는 성질 | 가용성만 답으로 나옴 |
| OS 스케줄링 | FCFS | 먼저 도착한 작업부터 처리하는 비선점 스케줄링 | HRN/SJF/SRT/RR만 나옴 |

### 9-4. 그래도 반복 출제 대비 (버리면 안 되는 영역)

아래 영역은 "빈칸 메우기"와 별개로 **매년 형태를 바꿔 반복 출제**되므로, 완벽한 암기 재현 수준으로 대비해야 한다.

| 영역 | 학습 포인트 |
|---|---|
| 디자인 패턴 | 이름·분류(생성/구조/행위)·뜻 세트 완벽 암기 |
| 결합도/응집도 | 강→약 순서 나열, 뜻 구분 |
| 테스트 커버리지 | 문장→분기→조건→변경조건결정 포함 관계 |
| OSI 7계층 | 계층별 역할, 계층별 프로토콜 매핑 |
| 라우팅 프로토콜 | RIP(거리벡터)/OSPF(링크상태)/BGP(경로벡터) 구분 |
| SQL 기본 문법 | SELECT/GROUP BY/HAVING/ALTER/CREATE INDEX 완전 재현 |
| 관계대수 기호 | σ(선택)/π(프로젝트)/⋈(조인)/÷(나눗셈) |
| 암호 알고리즘 분류 | 대칭키(DES/AES/ARIA/SEED/IDEA)와 비대칭키(RSA/ECC) 구분 |
| 프로그래밍 언어 | Java/C/C++/Python 코드 해석 출력형 문제 대비 |
| Linux 명령어 | chmod (r=4, w=2, x=1) 8진수 권한 계산, pwd/ls/cd/cp 기본 명령어 — **4회 이상 출제** |
| EAI | P2P / Hub&Spoke / Message Bus / Hybrid 4가지 유형 구분 암기 |
| 헝가리안 표기법 | 변수명 앞 자료형 접두어 표기 규칙 — 식별자 표기법 문제 유형 대비 |

### 9-5. 학습 전략 요약

1. **1순위**: 9-2의 ⭐⭐⭐ 구멍부터 뜻까지 암기 — 출제자가 의식적으로 메우러 올 확률이 가장 높은 항목들
2. **2순위**: 9-3의 ⭐⭐ 차상위 항목 뜻 암기
3. **3순위**: 9-4의 반복 출제군을 "변형 없이 완벽 재현" 수준으로 훈련
4. **별도 훈련**: 프로그래밍 언어 활용은 암기보다 코드 해석력이 관건이므로 기출 코드 반복 풀이

특히 **1NF, XSS, CSRF, INNER JOIN, TCP/UDP, 기밀성/무결성, SHA-256, 지연 갱신 회복 기법**은 카테고리 내 나머지가 다 출제된 상황에서 남은 핵심 빈칸이므로 **최우선 암기 대상**이다.

또한 **chmod, EAI, 헝가리안 표기법, Linux 기본 명령어**는 기출에서 이미 직접 답으로 나왔으나 기존 정리에서 누락되어 있었다. 반복 출제 가능성이 높으므로 9-4의 반복 출제 대비 목록과 함께 반드시 숙지해야 한다.
