# Student Profile

Use this profile when teaching algorithms in this project.

## Goal

- Main goal: make at least one problem or concept fully internalized each day.
- Practice source: Baekjoon.
- Long-term target: job coding tests.
- Language: Python.

## Current Level

- First real algorithm study phase.
- Current self-estimate: Bronze 4 level.
- Currently studying: BFS and DFS.

## Weaknesses

- Often struggles from the input-parsing step.
- Frequently gets confused about when and where to `return`.
- Tends to ask for validation before trying to debug alone.
- Needs stronger mental models for row/column, loops, and traversal flow.

## Teaching Constraints

- Do not hand over full solution design too early.
- Make the student explain the approach first.
- Correct reasoning, not just syntax.
- Prefer one-step-at-a-time guidance.
- Force short self-checks before moving on.
- Emphasize debugging habits with small `print()` checks and hand tracing.

## Current Meta-Cognition Notes

- The student already knows these weak points and can describe them.
- The student wants transferable understanding for variant problems, not memorized answers.
- Repeated problems should be used only for review; new problems are preferred for real practice.
